# MSXPLAYer Game Cartidge Adapter GUI 仕様メモ

## 目的

MSXPLAYer Game Cartidge Adapter を USB CDC のシリアルデバイスとして扱い、Python + `tkinter` で操作用 GUI を実装する。

この文書は、最初の実装方針と画面構成、通信の考え方を整理するためのもの。

## 前提

- Adapter は USB 接続時に CDC として認識され、シリアルポート経由で通信する
- 通信コマンド仕様は `docs/com_command.md` を正とする
- GUI は `tkinter` で実装する
- 外部ライブラリは最小限に抑える
- シリアル通信の既定値は `115200 / 8N1 / timeout 1秒` とする

## 画面の見た目

画面は HEX エディタに近い見た目とする。

- 左列: アドレス
- 中央列: HEX データ
- 右列: キャラクタ表示

表示は 64kB 全体を対象にし、通常の HEX エディタと同様に 1 行ごとに固定幅で並べる。
HEX の横幅は 16 バイト固定とする。
キャラクタ表示は印字可能 ASCII のみそのまま表示し、それ以外は `.` とする。

## 起動オプション

シリアルデバイスはコマンドライン引数で指定できるようにする。

例:

```text
--device=/dev/ttyACM0
```

`--device` は起動時に自動接続するためではなく、シリアル選択ダイアログでの事前選択に使う。

## 起動時フロー

起動直後はシリアル未接続の状態で開始する。

- HEX 画面は 64kB 全体を `00` で初期表示する
- 接続は `設定 -> シリアル` またはツールバーのシリアルボタンから行う

シリアルデバイスを開いた直後は以下の順で処理する。

### 1. `HVER` 実行

- Adapter に対して `HVER` を送信する
- バージョン情報が取得できることを確認する
- 取得できなかった場合はエラー表示を出し、未接続状態に戻る

### 2. 64kB 全体の読出し

- 64kB 全体をシリアル経由で読み出す
- 読出し対象はバッファ全体の `0000h` から `FFFFh` までとする
- 初回読出し時は以下のシーケンスで行う

```text
SPON
1 秒待つ
SMTR,0000,10000,0,1
BSND,
```

- `SMTR` で Adapter 側バッファに転送し、その後 `BSND` で PC へ受信する
- `10000` は 64kB 全体を意味する長さとして扱う
- 起動後はスロット電源を ON のまま維持する
- 読出し完了まで GUI は待機状態になる

### 3. HEX 画面へ表示

- 読み出した 64kB の内容を HEX 画面へ反映する
- アドレス、HEX データ、キャラクタ表示を同時に更新する
- 読出し前の空画面ではなく、ロード進行が分かる表示を用意することが望ましい

## 推奨構成

### 必須

- Python 3.10 以上
- `tkinter`
- `pyserial`

### 推奨理由

- `tkinter` は標準ライブラリなので GUI 追加コストが低い
- シリアル通信は標準ライブラリだけでは扱いづらいため、`pyserial` の採用が最も実用的
- 依存を増やさずに、Windows / macOS / Linux のクロスプラットフォーム対応がしやすい

### 追加候補

- `pyinstaller`
  - 配布用の単体実行ファイルを作る段階で必要なら採用する
- `logging`
  - 標準ライブラリで十分。専用ライブラリは不要

## GUI の基本方針

GUI は HEX エディタのような見た目にする。
上部にメニューバーとツールバーを配置する。

### メニュー

- ファイル
  - 保存
  - 終了
- 設定
  - シリアル
- デバッグ
  - 手動実行
  - ログ

### ツールバー

- 保存
- シリアル
- 右端に `00`, `40`, `80`, `C0`, `FF` のジャンプボタンを配置する
- 各ボタンはそれぞれ `0000`, `4000`, `8000`, `C000`, `FFFF` へスクロールする

### シリアル選択

- `設定 -> シリアル` とツールバーの `シリアル` は、シリアルデバイス選択ダイアログを開く
- ダイアログには `pyserial` により検出したシリアルデバイス一覧をドロップダウンリストで表示する
- 一覧から 1 つ選択して開けるようにする
- 現在使用中のデバイスが一覧にあれば初期選択する
- シリアルデバイスを開いた直後に初期化シーケンスを実行する
- デバイス切替時は新しい接続で `HVER` と 64kB 再読込を行い、成功時だけ切り替える
- 未保存の編集内容がある場合は、切替前に確認を出す

### 直接書き込み

- HEX 編集で 1 バイト確定時に即座に 1 バイト書き込みを行う
- 1 バイト書き込み後は `SMTR` と `BSND` により 64kB 全体を再読込し、HEX 画面を再表示する
- 直接書き込み後の再読込では `SPON`、待機、`SPOFF` は行わない

### 保存ダイアログ

- `保存` を押すと保存ダイアログを表示する
- ファイル名を指定できる
- 保存するアドレス範囲を指定できる
- 開始アドレスはテキスト入力可能なドロップダウンとする
- 終了アドレスはテキスト入力可能なドロップダウンとする
- 開始アドレス候補は `0000`, `4000`, `8000`, `C000` とし、既定値は `0000`
- 終了アドレス候補は `3FFF`, `7FFF`, `BFFF`, `FFFF` とし、既定値は `FFFF`
- 保存時は指定範囲を含むデータだけをファイルへ書き出す

### 保存時の入力検証

- 開始アドレスと終了アドレスは 16 進数のみ受け付ける
- `0000` から `FFFF` の範囲外はエラーとする
- 開始アドレスが終了アドレスを超えていたらエラーとする

## 画面構成案

### 1. HEX 表示エリア

- アドレス列
- HEX データ列
- 文字表示列
- HEX 表示エリア上部に固定ルーラを表示する
- 初期段階では閲覧専用とする
- 1 行 16 バイト固定

表示形式の基本イメージ:

- 1 行 16 バイト
- 左に 4 桁アドレス
- 中央に 16 バイトの HEX
- 右に ASCII 相当の可視文字
- 最上部に `ADDR  +0 +1 ... +F  0123456789ABCDEF` のルーラを常時表示する
- ルーラは縦スクロールの影響を受けず、横スクロール時は HEX 本体と同期する

キー入力やカーソル移動イベントをフックし、アドレスやスペースにカーソルが移動できないようにする。
HEX 部分では `0-9`, `a-f`, `A-F` 以外のキャラクタ入力は無視する。
変更した箇所は赤字にする。
- `PageUp` / `PageDown` でそれぞれ `-0100h` / `+0100h` 移動する
- `Ctrl+PageUp` / `Ctrl+PageDown` でそれぞれ `-0800h` / `+0800h` 移動する

書き込み操作をするまではオンメモリで編集するだけに留める。

### 2. 手動実行ウィンドウ

- `デバッグ -> 手動実行` を選ぶと手動実行ウィンドウを開く
- 手動実行ウィンドウに `Command` 入力欄と実行ログ欄を配置する
- 手動実行ウィンドウのログには、そのウィンドウから実行したコマンド結果のみを表示する
- メイン画面での起動時アクセスログや自動読出しログは表示しない

### 3. アクセスログダイアログ

- `デバッグ -> ログ` を選ぶとアクセスログを別ダイアログで開く
- アクセスログダイアログには adapter との送受信ログを表示する
- メイン画面には `Command` と `Log` を配置しない

## 通信仕様の扱い方

### コマンド送信

- GUI で入力された 1 行をそのまま送信する
- 改行は `CR`, `LF`, `CRLF` の受け側仕様差異を吸収できるようにする
- 送信後は応答をログへ表示する

### 応答受信

- 文字列応答は行単位で受信し、ログ欄へ表示する
- `OK` / `FAIL` は結果判定として扱う
- `BSND` などのバイナリ応答は文字列と混在するため、バイナリ受信モードを分けて扱う

### バイナリ転送

以下は「コマンド送信後に固定長のバイナリ本体を送る / 受ける」処理として実装する。

- `BRCV`
- `BSND`
- `SMTR`
- `SMTW`
- `IOTR`
- `IOTW`

実装上は以下を分離する。

- コマンド入力層
- バイナリ入出力層
- 画面表示層

## 実装上の注意

### 1. UI のフリーズ防止

シリアル待ち受けは GUI スレッドで直接行わず、別スレッドか `after()` ベースでポーリングする。

### 2. 送受信の競合回避

- 送信中に別コマンドを重ねない
- バイナリ転送中は入力欄を一時的に無効化する
- 応答待ちタイムアウトを設ける

### 3. ログの可読性

- 送信行は `>` プレフィックス
- 受信行は `<` プレフィックス
- バイナリは HEX 表示とファイル保存を切り替えられるようにする

### 4. コマンドの前処理

- 大文字小文字を気にせず入力できるようにする
- 16進数入力の補助を入れる
- 省略可能引数は GUI 側で空欄を許可する

### 5. 初期読出しの失敗時処理

- `HVER` に失敗した場合、以降の起動処理は行わない
- エラーは画面上で明示する
- 再試行の要否は初期段階では未定義とする

## 実装順

### 1. 起動と読出し

- `--device` によるシリアルデバイス指定
- `HVER` による起動時疎通確認
- 64kB 全体の読出し
- HEX エディタ風の表示
- 手入力コマンド送信
- ログ表示

### 2. HEX編集

以下をまとめて「HEX編集」タスクとする。

#### 2-1. HEX 編集の基盤

- 表示専用の HEX 領域を編集可能な構造へ切り替える
- カーソル位置とバッファ位置を相互に対応付ける
- HEX 部分だけ編集可能にする
- アドレス列とキャラクタ列は直接編集不可とする
- `0-9`, `a-f`, `A-F` 以外の入力は無視する
- 変更バイトは赤表示にする

#### 2-2. 編集状態の管理

- 元データ 64kB と編集中データ 64kB を分けて保持する
- 変更済みバイト集合を管理する
- 未保存変更の有無を判定できるようにする

#### 2-3. キャラクタ列の連動更新

- HEX 編集時に右側のキャラクタ表示を即時更新する
- ASCII 列からの直接編集は行わない

### 3. 次タスク

以下は「HEX編集」の後に進める別タスクとする。

- 明示操作によるデバイスへの一括書き戻し
- `SCHK`、`HVER` などの補助操作
- 手動コマンド履歴
- 編集ナビゲーション

## 依存ライブラリの提案

最小構成では以下で十分。

- `pyserial`

必要になった場合のみ追加する候補。

- `pyinstaller`:
  配布ビルド用
- `ttkbootstrap` などのテーマ系:
  見た目を整える用途。ただし依存増加があるため初期段階では非推奨

## 実装方針の結論

- GUI は `tkinter`
- 通信は `pyserial`
- 余計な外部依存は増やさない
- まずは「接続・送信・受信ログ・基本コマンド」の最小実装から始める
